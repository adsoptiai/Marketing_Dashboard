# 專案開發規格書：醫美短影音市場情報與 AI 分析系統 (Local Web App)

## 1. 專案概述
本專案旨在建立一個本地部署的視覺化 Web 介面，用於自動化抓取特定醫美關鍵字的 YouTube Shorts 數據，透過 AI Agent 進行資料分析與報告生成，並提供串接 LINE Messaging API 的推播功能。系統需包含手動觸發與每週自動化排程功能。

## 2. 技術選型建議
* **前端與 Web 框架**: `Streamlit` (Python) - 適合快速建構數據儀表板與 AI 互動介面。
* **資料處理與視覺化**: `Pandas`, `Plotly` 或 `Streamlit native charts`。
* **YouTube 爬蟲/API**: `google-api-python-client` (YouTube Data API v3)。
* **AI Agent**: 串接 OpenAI / Gemini API 進行數據摘要與報告生成。
* **通訊整合**: `line-bot-sdk`。
* **排程管理**: `APScheduler` (在背景執行每週自動化任務)。
* **資料儲存**: `SQLite` (本地輕量化關聯式資料庫) 或純本地 `.csv` / `.json` 落地。

## 3. 系統核心功能與 UI 佈局 (Streamlit介面設計)

### 區塊 A：側邊欄 (Sidebar) - 系統設定與狀態
* **環境變數設定區**: 供使用者輸入或檢查 API Keys (YouTube API, LLM API, LINE Channel Access Token, LINE User ID)。
* **排程狀態指示器**: 顯示目前「每週自動排程」是否啟用，以及下次執行的預計時間。

### 區塊 B：主畫面分頁 1 - 關鍵字與數據抓取 (Data Scraper & Dashboard)
* **關鍵字展示區**: 
    * 讀取本地 CSV 檔 (`yt-海外華人拉皮都指名要找...csv`)。
    * 使用 `st.multiselect` 或 `st.dataframe` 呈現並允許使用者預覽目前監控的核心字（如：8字線雕、嘴邊肉下垂、華人拉皮推薦等）。
* **手動觸發區**:
    * 按鈕：`[ ▶️ 開始抓取最新數據 ]`。點擊後觸發 YouTube API 爬蟲流程（搜尋 -> 過濾 60 秒內 Shorts -> 獲取觀看/按讚數）。
    * 進度條：使用 `st.progress` 顯示抓取進度，避免 API 請求過快。
* **數據可視化區**:
    * 抓取完成後，呈現**結構化資料表**（影片標題、連結、觀看數、發佈時間）。
    * 呈現**圖表**：例如「各關鍵字總觀看數長條圖」或「過去一週熱門 Shorts 圓餅圖」。

### 區塊 C：主畫面分頁 2 - AI 分析報告與 LINE 發送 (Agent Report & Notify)
* **Agent 報告生成介面**:
    * 讀取「區塊 B」抓取好的結構化數據。
    * 按鈕：`[ 🤖 生成市場情報報告 ]`。
    * 文字展示區：動態打字機效果呈現 Agent 產出的 Markdown 報告（包含：趨勢摘要、爆款影片分析、診所/Sleep ABC 社群選題建議）。
* **LINE 推播功能**:
    * 按鈕：`[ 📲 將報告發送至 LINE ]`。
    * 點擊後透過 LINE Messaging API 將報告推播至指定的 LINE 群組或個人帳號，並顯示成功提示。

### 區塊 D：主畫面分頁 3 - 自動化設定 (Automation & Scheduling)
* **每週自動化開關**:
    * 切換按鈕 (Toggle/Checkbox)：`[ 🔄 啟用每週自動執行 ]`。
    * 設定執行時間：如下拉選單選擇「每週一 早上 09:00」。
* **自動化流程定義**:
    當排程觸發時，系統需在背景依序靜默執行：`讀取關鍵字` -> `抓取一週數據` -> `Agent 生成報告` -> `自動發送 LINE 訊息`。

## 4. Agent 開發實作步驟指示 (Action Plan)

請 Agent 依照以下步驟依序開發：

**Step 1: 專案初始化與套件安裝**
* 建立 `requirements.txt` 並安裝所需套件 (`streamlit`, `pandas`, `google-api-python-client`, `apscheduler`, `line-bot-sdk` 等)。

**Step 2: 核心功能模組化 (分離業務邏輯與 UI)**
* 開發 `youtube_service.py`: 實作 API 兩階段撈取邏輯（限制時間區間為過去 7 天，篩選 duration <= 60s）。
* 開發 `llm_agent.py`: 實作將 DataFrame 轉為 Prompt，並呼叫 LLM 產出專業醫美市場分析的邏輯。
* 開發 `line_service.py`: 實作 Push Message 功能。

**Step 3: 建構 Streamlit 介面 (`app.py`)**
* 實作區塊 A ~ D 的 UI 佈局。
* 整合 Step 2 的模組，確保點擊按鈕時能正確觸發功能並更新 Session State。

**Step 4: 整合 APScheduler 背景排程**
* 在 `app.py` 中初始化 `BackgroundScheduler`。
* 將「自動化流程」封裝為單一函數，綁定至排程器中，確保 Streamlit 伺服器運行時能定時觸發。

## 5. 錯誤處理與日誌紀錄
* API 配額限制防護：在 YouTube API 請求中加入適當的 Delay (`time.sleep`)。
* 需在終端機輸出清晰的 Logging，以便追蹤背景排程的執行狀態與錯誤原因。
